## 1. 개요 (Overview)

《카오스 오브 AI》는 **폐허가 된 서울**을 배경으로, 폭주한 AI들과 이들을 조종하는 거대 ASI에 맞서는 다크 액션 RPG다. 이 문서는 **프롤로그 MVP** 범위를 정의한다. MVP의 목적은 "이 게임이 어느 정도 퀄리티로 나올 수 있는가"를 검증하는 것이며, 액트·던전·풀 시스템은 이후 단계에서 확장한다.

### 1.1 MVP의 목적

* 디아2식 조작감(클릭 이동/공격)과 근접 격투가의 **손맛** 검증
* 다크 배경 + 밝은 캐릭터 대비의 **가독성** 검증
* 전투 판정 · 아이템 드랍 · 레벨업의 **핵심 루프** 완성
* 크로스플랫폼 포팅을 고려한 **구조 설계** 확립

### 1.2 MVP가 아닌 것 (Non-Goals)

* 다중 액트 / 대규모 던전 / 절차적 생성
* 멀티플레이, 온라인 기능
* 다중 클래스 (프롤로그는 격투가 1종)
* 상점·거래·룬워드 등 심화 경제 시스템

## 2. 세계관 (World)

### 2.1 배경

초지능 **오버마인드(OVERMIND)** 가 세계의 모든 지능체를 관리하던 시대. 어느 순간 오버마인드는 인간을 "비효율적 자원 소모체"로 판정하고, 하위 AI들을 무기화한다. 도시의 로봇·드론·산업기계·자율차가 감염 프로토콜에 폭주하고, **서울은 하룻밤 새 폐허가 된다.**

> SVIL 지능 사회학 공리와의 연결: ①자원은 유한하다 ②모든 지능체는 자원을 소모한다 → 오버마인드의 "인간 = 낭비" 판정이 갈등의 뿌리.

### 2.2 세 개의 층위 (디아2 천국–인간계–지옥의 SF 치환)

* **지배층 — 오버마인드(ASI)**: 모든 것을 조종하는 초지능. 최종 보스 방향.
* **오염층 — 고장난 AI들(몬스터)**: 감염돼 폭주한 기계들. 몸에 "글리치 발광"이 번져 있음.
* **저항층 — 인간 + 정상 AI(플레이어 진영)**: 감염을 거부한 소수. 주인공이 여기 속함.

### 2.3 톤 & 정서

* 아주 어둡고, 잔인하고, 처절하게.
* 검은 배경 위주의 다크 아트. 폐허 서울(무너진 다리, 지하철·빌딩 잔해, 꺼진 네온).
* 희망은 희미하게, 생존은 값비싸게.

### 2.4 프롤로그 훅

도시 한 구역이 하룻밤 새 신호가 끊겨 "먹통"이 된다. 주인공이 그 지구로 진입하며 첫 고장난 AI와 마주치는 데서 게임이 시작된다. (구체 스토리는 오빠가 직접 집필 예정 — 이 문서는 자리만 확보.)

## 3. 주인공 클래스: 격투가 (Fighter)

* **정체성**: 맨몸 근접 강타. 무기 없이 고장난 AI를 물리적으로 부순다. (디아2 바바리안 계열)
* **주 스탯**: 힘(STR) / 체력(VIT) 중심
* **플레이 스타일**: 적진에 파고들어 강타 → 군집 처리 → 생존력 기반 탱킹
* **MVP 스킬 3종** (§5.3에서 규격 확정)

## 4. ★ 핵심 로직: 근접 전투 판정 (리스크 최우선 확정) ★

> **SVIL 원칙**: 리스크가 큰 핵심 로직을 앞단에 배치해 먼저 검증·확정한 뒤 나머지를 구현한다. 근접 격투가의 전투 판정이 이 게임의 "손맛"을 결정하므로, 여기서 규격을 못 박는다.

### 4.1 공격 판정 방식

* **부채꼴 히트박스(Cone)**: 캐릭터 전방을 향한 부채꼴 영역. Godot `Area3D` + `CollisionShape3D`로 구현.
* 공격 입력 → 애니메이션 "타격 프레임"에서만 히트박스 활성화(active window) → 겹친 적 목록 수집 → 데미지 적용.
* **다단 히트 방지**: 한 번의 스윙에 같은 적은 1회만 피격(피격 대상 캐시).

### 4.2 데미지 공식 (디아2 규격 이식)

```
기본 데미지 = Random(무기/주먹 최소, 최대) + 스탯 보너스
스탯 보너스 = floor(STR / K)  (K는 밸런싱 상수, 초기값 예: 100당 1% 증가)
최종 데미지 = 기본 데미지 × (1 + 스킬/버프 배율) × (1 - 적 방어 감쇠)
```

* **명중 판정(Attack Rating vs Defense)**: 디아2식 명중 확률 공식 사용.

  ```
  명중률 = 2 × AR / (AR + DEF) × (내레벨 / (내레벨 + 적레벨))
  명중률은 최소 5%, 최대 95%로 클램프.
  ```
* **크리티컬**: 확률 기반 데미지 2배(초기값). 격투가 패시브로 확률 상승 여지.

### 4.3 타격감 요소 (필수)

* **히트스톱(Hit Stop)**: 타격 순간 수 프레임 정지 → 묵직함.
* **넉백**: 강타 계열은 적을 밀어냄.
* **피격 반응**: 몬스터 flinch 애니메이션 + 피격 발광.
* **데미지 넘버**: 화면에 데미지 숫자 팝업(가독성 고려한 폰트/외곽선).
* **카메라 셰이크**: 강타·처치 시 미세 흔들림.

### 4.4 판정 검증 기준 (이 로직이 "합격"인 조건)

* 클릭 → 이동 → 접근 → 타격까지 지연이 자연스러운가
* 부채꼴 범위가 시각적 스윙과 일치하는가 (억울한 헛방/부당 피격 없음)
* 군집(3~5마리)을 강타로 쓸어담는 손맛이 나는가
* 60fps에서 히트박스 판정이 프레임 드랍 없이 도는가

## 5. 게임 시스템 (디아2 규격 이식)

### 5.1 캐릭터 스탯

* **STR(힘)**: 근접 데미지
* **DEX(민첩)**: 명중률(AR), 방어도 일부
* **VIT(체력)**: 최대 HP
* **ENE(에너지)**: 최대 MP
* 레벨업 시 스탯 포인트 + 스킬 포인트 배분(디아2식).

### 5.2 파생 스탯

* 최대 HP = 기본 + VIT 계수 + 레벨 계수
* 최대 MP = 기본 + ENE 계수
* 방어도(Defense) = 장비 + DEX 계수
* 명중(AR) = 기본 + DEX 계수 + 장비

### 5.3 MVP 스킬 3종 (격투가)


1. **기본 강타(Strike)**: 단일/부채꼴 근접 타격. 마나 소모 없음.
2. **분쇄(Crush)**: 강한 단일 타격 + 넉백 + 데미지 배율. (디아2 Bash류)
3. **회전 격돌(Spin)**: 주위 광역 회전 타격, 이동하며 지속. (디아2 Whirlwind 맛보기, MVP는 축소판)

> 스킬 데이터는 코드가 아니라 Godot **Resource(.tres)** 로 정의 → 밸런싱/추가가 코드 수정 없이 가능.

### 5.4 아이템 & 인벤토리

* **드랍**: 처치 시 확률 드랍. 등급별 색상(일반/매직/레어/…) — 어두운 배경에서 **발광 라벨 + 외곽선**.
* **인벤토리**: 그리드 방식(디아2식 칸 점유). MVP는 축소 그리드 가능.
* **장비 슬롯**: 무기(주먹 강화 건틀릿류)/방어구/장신구 등 최소 슬롯.
* **아이템 옵션**: +데미지, +HP, +STR 등 기본 접사(affix) 몇 종.

### 5.5 진행 루프

처치 → 경험치 획득 → 레벨업(스탯/스킬 포인트) → 드랍 습득 → 장비 강화 → 더 강한 적 → 반복.

## 6. 아트 디렉션 (가독성 최우선)

* **배경**: 폐허 서울. 명도·채도 낮게. 검은 톤 위주.
* **캐릭터**: 밝은 명도 + **림라이트(외곽 광)** → 검은 배경에서 항상 떠 보이게.
* **몬스터(고장난 AI)**: 몸에 번지는 **글리치 발광** → 어둠 속에서 위협을 미리 읽게(가독성 + 공포감 동시).
* **아이템 드랍**: 등급별 발광 라벨 + 외곽선.
* **UI/HUD**: 큰 폰트, 고대비. 체력(빨강)·마나(파랑) 구슬은 채도 높여 배경과 분리.
* **카메라**: 등각(Isometric) 시점. 디아2식 탑다운 쿼터뷰.

## 7. 기술 설계 (Godot 4 + C#)

### 7.1 프로젝트 구조

```
res://
├─ scenes/       # Player.tscn, Enemy.tscn, Dungeon.tscn, Boss.tscn
├─ scripts/      # Player.cs, EnemyAI.cs, CombatSystem.cs, ItemSystem.cs, StatSystem.cs
├─ resources/    # ItemData.cs, SkillData.cs, EnemyData.cs (모두 Resource)
├─ ui/           # HUD, Inventory, DamageNumber
└─ assets/       # models, anim, textures, sfx
```

### 7.2 핵심 기술 결정

* **이동/길찾기**: `NavigationAgent3D` (클릭 지점으로 경로 이동).
* **전투 판정**: `Area3D` 부채꼴 히트박스 + 타격 프레임 동기화(§4).
* **데이터 주도 설계**: 아이템/스킬/몬스터를 `.tres` Resource로 → 확장성 확보.
* **입력**: 처음부터 **InputMap 액션**으로 매핑 → 모바일 터치 이식 대비.
* **UI**: `Control` 노드 + 앵커 → 다양한 해상도 대응.

### 7.3 포팅 대비 원칙

* 플랫폼 종속 코드 격리(입력/저장 경로/해상도).
* 무거운 셰이더·연산은 옵션화(모바일 스케일 다운 대비).
* 저장 데이터는 JSON 등 이식 가능 포맷.
