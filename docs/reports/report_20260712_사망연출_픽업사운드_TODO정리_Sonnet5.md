## 1. 배경

코드에 남아있던 `TODO(Sonnet)` 주석을 훑어 처리 가능한 항목을 마무리했다. 정식 애니메이션(스켈레톤·클립)은 아트 자산이 필요해 범위 밖이지만, **코드만으로 가능한 절차적 연출**은 구현했다.

## 2. 몬스터 사망 연출

- `EnemyAI.Die()`: 기존엔 처치 즉시 `QueueFree()`로 그냥 사라졌다. 이제 충돌은 즉시 해제하고, `Tween`으로 메시를 0으로 축소 + 알파를 0으로 페이드(≈0.35초)한 뒤 소멸한다.
- 경험치·드랍 지급(`GrantRewardsToPlayer`)은 연출과 무관하게 **즉시** 확정(판정에 지연 없음, 시각 효과만 지연).
- `flinch 애니메이션` TODO는 주석을 갱신해 "아트 자산 필요, 현재는 피격 발광+넉백으로 대체"로 명확히 함(이미 v0.6.0에서 발광은 구현됨).

## 3. 아이템 습득 사운드

- `pickup.wav`(합성 2음 상승 칭) 추가, `ItemPickup.OnBodyEntered`에서 재생.
- 픽업 노드 자체는 습득 즉시 사라지므로, 사운드는 부모(현재 씬)에 독립 `AudioStreamPlayer3D`를 심어 재생 후 스스로 정리(기존 `Player.PlaySfx` 패턴과 동일).

## 4. 테스트 타이밍 보정 (연출 지연의 부작용 수정)

사망 연출이 QueueFree를 지연시키면서, "처치 직후 노드가 즉시 사라지는지"를 짧게(2프레임) 확인하던 `CombatSmokeTest`가 깨졌다. **"노드가 사라졌는가"는 연출 구현 디테일**이므로, 판정 기준을 `State == EnemyState.Dead`(즉시 확정되는 시맨틱 상태)로 옮기고, 대기 시간을 30프레임으로 늘려 실제 `QueueFree`까지 끝까지 확인하도록 재작성했다. `DungeonSmokeTest`는 `Died` 시그널(즉시 발생)만 확인해 원래부터 영향 없었다.

## 6. 남은 것 (사실상 전부 범위 밖 항목만 남음)

- **정식 아트 에셋**(3D 모델·스켈레톤 애니메이션·정식 사운드·이펙트 텍스처) — 생성형 파이프라인 또는 외주 결정 필요, 코드로 대체 불가능한 영역.
- **스토리/대사** — PRD가 "오빠 집필 예정"으로 명시.
- **그리드 인벤토리 시각화** — 현재 목록형 패널로 기능은 충족, 시각 개선은 아트 패스와 함께가 효율적.
