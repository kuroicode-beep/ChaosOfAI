# 완료 보고서 — M5 프롤로그 던전 + 폴리싱 (v0.5.0 → v0.6.0)

- 작성: Fable 5 (Claude Code) · 2026-07-10
- 선행: [검수 수정 + 실행 파일 빌드 보고](report_20260710_검수수정_실행파일빌드_Opus.md) (v0.4.x)
- 지시: "프로젝트 계속 진행해"

## 1. v0.5.0 — M5 프롤로그 던전

PRD §8 마일스톤의 M5(프롤로그 던전: 방 5~6개 + 보스방)를 구현했다.

- **DungeonBuilder** (`scripts/core/DungeonBuilder.cs`): ASCII 맵 데이터 주도 던전 생성.
  - 기호: `#`=벽, `P`=플레이어, `E`=글리치 드론, `W`=녹슨 보행기, `B`=보스. 셀 1칸 = 2m.
  - 바닥/벽(StaticBody3D, world 레이어)/스폰을 런타임 생성 — 레이아웃 수정이 문자열 몇 줄 편집으로 끝남(§7.2 데이터 주도 원칙의 연장).
  - 구성: 시작방 → 복도 → 전투방들 → 보스방(우측 끝). 맵 밖은 확장 바닥으로 void 노출 방지.
- **보스**: "오버마인드의 사도"(`overmind_herald.tres`) — Lv5, HP 크게, 드랍 100%, 스케일 1.7배.
- **클리어 흐름**: 보스 `Died` 시그널 → HUD "프롤로그 클리어" 오버레이 + R 재시작.
- **시야 판정(LOS)**: 몬스터가 벽 너머 플레이어에게 어그로 끌리던 문제를 레이캐스트로 차단.
- 시작 씬을 `Dungeon.tscn`으로 변경(기존 `Main.tscn`은 아레나 테스트용으로 유지).

## 2. v0.6.0 — 폴리싱 2건

- **인벤토리 패널(I 키)**: 습득 아이템 목록 — 등급 색상 + 텍스트 라벨 병행(전역 접근성 규칙), 접사 요약(+데미지/+HP/+힘/+방어/+명중). 픽업 시 실시간 갱신.
- **몬스터 피격 발광(flinch, §4.3)**: 명중 시 0.12초 흰색 emission 플래시. 개체별 머티리얼 복제로 다른 몬스터와 간섭 없음.

## 3. 검증 (전부 이 세션에서 실행)

| 항목 | 결과 |
|---|---|
| `dotnet build` | 오류 0 / 경고 0 |
| 회귀 테스트 6종 (로직 9 + 진행 14 + 전투통합 + 사망 + 이동 + **던전/승리 신규**) | **전부 PASS** |
| 던전 렌더 캡처(실제 창) | 방·복도·벽·적 배치·HUD 확인 |
| 인벤토리 I키 주입 캡처 | 패널 표시 확인 |
| exe 부팅(v0.6.0, 던전 씬) | exit 0 |

신규 테스트 `DungeonSmokeTest`: 던전 스폰 검증 + 보스 강제 처치 → 클리어 패널 표시까지 엔진에서 확인.
