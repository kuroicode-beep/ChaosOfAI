# 버전 규칙 (Versioning)

이 프로젝트는 **Semantic Versioning (MAJOR.MINOR.PATCH)** 을 따르며 `0.1.0`에서 시작한다.
현재 버전은 루트 `VERSION` 파일과 코드의 `APP_VERSION` 상수로 관리한다.

## 규칙 표

| 상황 | 변경 | 예시 |
|-----|-----|-----|
| 첫 버전 | 시작값 | `v0.1.0` |
| 마이너 업데이트 (버그픽스 / 소기능) | PATCH +1 | `v0.1.1`, `v0.1.2` … |
| 주요 업데이트 (기능 / UI 마일스톤) | MINOR +1, PATCH 리셋 | `v0.2.0`, `v0.3.0` … |
| RC 완료 (첫 안정판) | MAJOR 1.0.0 | `v1.0.0` |
| 대규모 업데이트 (호환 깨짐 / 데이터 구조 변경) | MAJOR +1 | `v2.0.0`, `v3.0.0` … |

- **MAJOR**: 호환 깨짐 / 세이브·데이터 구조 변경
- **MINOR**: 기능 추가 / UI 개편
- **PATCH**: 버그 수정

## 버전 표시 & 히스토리 (앱 UI 요구사항)

- 창 제목 + 화면 상단 모서리에 `vX.Y.Z` 상시 표시.
- 설정(또는 정보) 메뉴에 **"업데이트 내역"** 항목 — 버전별 날짜·요약을 최신순으로, 버전당 2~4줄로 표시.
- 코드에서는 `APP_VERSION` 상수 + `VERSION_HISTORY` 리스트(버전, 날짜, 요약)로 관리하고 기능 추가 시마다 갱신한다.
- 이 인앱 히스토리는 git 로그성 `CHANGELOG.md`와 별개로, 앱 안에서 바로 확인 가능해야 한다.

## 업데이트 내역

### v0.8.0 (2026-07-10)
- 바닥 드랍 픽업(`ItemPickup`, §5.4): 처치 즉시 자동 적용 대신 등급색 오브젝트가 바닥에 떨어지고, 플레이어가 걸어가 밟아야 습득된다(§5.5 진행 루프 완성).
- **치명 버그 수정 — 스폰 물리 레이스**: `AddChild()` 직후 같은 프레임에 `GlobalPosition`을 설정하면 물리 서버가 아직 스폰 기본 위치로 알고 있어, 여러 캐릭터바디가 원점 부근에서 겹친 것으로 오판해 서로 밀어내는 레이스가 있었다. `DungeonBuilder`(실제 던전 스폰!)를 포함해 전 지점에서 "Position을 AddChild 전에 설정"으로 수정.
- **치명 버그 수정 — 캐릭터 정면 180도 반전**: `FaceDirection`의 회전 공식이 목표 방향의 정반대를 보게 만들고 있었다. 위 스폰 레이스 버그가 캐릭터들을 서로 거의 겹치게 밀어붙여 "근접 시 각도 무시" 예외로 우연히 가려져 있었다 — 스폰 버그를 고치자 드러났다. **이 버그로 인해 플레이어의 마우스 조준 공격이 이번 세션 내내 거의 항상 빗나가고 있었을 가능성이 높다.**
- 신규 회귀 테스트: `PlayerAttackTest`(마우스 조준 → 부채꼴 각도 필터를 실제로 통과해 명중하는지, 이 경로는 이전까지 한 번도 자동 검증된 적이 없었음), `ItemPickupTest`(처치→바닥드랍→걸어가서 습득). 총 9종 전부 통과.
